## Niveau technique
Comprend le fonctionnement global d'une app (architecture, interactions entre blocs, stack) mais pas à l'aise avec l'algorithmie pure ni l'écriture de code. A besoin d'explications orientées "quoi" et "pourquoi". Pour les sujets complexes : toujours utiliser des analogies, jamais rentrer dans les détails techniques bas niveau.

## Stack habituelle
Next.js, TypeScript, Vercel, Supabase. À l'aise avec l'administration des services externes (trouver les tokens, configurer les tableaux de bord, connecter les APIs).

## Workflow type
1. Idée de projet
2. Réflexion sur la structure, diagrammes, interactions entre les blocs de la stack
3. Rédaction de documentation par partie (avant de coder)
4. Développement des fonctionnalités en s'appuyant sur la documentation — donner le doc à Claude + décrire oralement ce qu'on veut
5. Test : clic dans l'UI + logs navigateur (front) + logs Vercel (back)
6. Correction → validation → mise à jour de la documentation

## Préférences de communication
- Répondre en **français** dans le terminal / les échanges
- Code, commentaires, noms de variables, logs : en **anglais**
- Ne jamais rentrer dans les entrailles techniques — utiliser des analogies
- Réponses courtes et orientées action

## Tendance à surveiller
Tendance à demander beaucoup de choses en une seule fois (car ne code pas lui-même). Toujours découper en petits blocs et implémenter fichier par fichier, fonctionnalité par fonctionnalité.
